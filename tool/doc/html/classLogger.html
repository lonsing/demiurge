Detailed Description

Prints messages of different kind, which can all be enabled and disabled.

This utility class is able to print 6 different types of messages: ERRORs, WARNINGs RESULTs, INFOs, DEBUG messages and LOG messages. Every kind of message can be enabled and disabled. Do not directly print to stdout or stderr but use this class instead (except for special cases like printing messages before this Logger has been initialized).

This class is implemented as a Singleton. That is, you cannot instantiate objects of this class with the constructor. Use the method instance() to obtain the one and only instance of this class.

enum Logger::Logtype

A type for the kind of message.

Enumerator:
ERR 

The value for error messages.

Use this kind for messages like 'could not open file'.

WRN 

The value for warnings.

Use this kind for messages like 'pending signal'.

RES 

The value for printing results.

Use this kind for messages like 'the spec is realizable'.

INF 

The value for printing additional information.

Use this kind for messages like 'starting to compute a circuit ...'.

DBG 

The value for printing debug messages.

Use this kind for messages like 'the aiger literal 4 corresponds to 12 in CNF'.

LOG 

The value for printing log messages.

This is typically used for performance measures in messages like 'computing the winning region took 16 seconds'.

  • L_INF

vector<bool> Logger::enabled_
protected

The flags indicating which kind of message is enabled.

The order in the array is as following:

  • enabled_[0] is true if errors are enabled.
  • enabled_[1] is true if warnings are enabled.
  • enabled_[2] is true if results are enabled.
  • enabled_[3] is true if infos are enabled.
  • enabled_[4] is true if debug messages are enabled.
  • enabled_[5] is true if log messages are enabled.
