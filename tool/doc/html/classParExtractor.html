Detailed Description

A parallelized version of the LearningImplExtractor.

This class implements a parallelized version of the LearningImplExtractor. Take a look at the comments in the LearningImplExtractor to see what it does. The parallelization is very simple at this point. It is merely a portfolio parallelization where different methods run in different threads. Closer interaction is left for future work.

Takes a circuit defining the control signals and inserts it into the spec.

This method takes as argument an aiger circuit of the following form:

  • The circuit is combinational, i.e., has no latches.
  • The inputs of the circuit are: the uncontrollable inputs from the spec (in the order in which they appear in the spec) followed by the state-signals (the output of the latches in the spec, in the order in which they appear in the spec).

volatile int ParExtractor::extr_command_

An integer number encoding the current command to the circuit extractor workers.

There are two possible value.

  • EXTR_GO = 0: means that the circuit extractor threads should continue.
  • EXTR_STOP = 1: means that the circuit extractor threads should stop.

All worker-threads poll this flag from time to time. If the value is still EXTR_GO they continue to work. If they see that the values is EXTR_STOP they stop, assuming that some other thread has already found the solution. If one thread finds a solution, it sets this flag to EXTR_STOP.
